# zig-js

A pure-Zig JavaScript engine with a JavaScriptCore-shaped C API. No JSC, V8, or external C libraries; no compatibility promises before stabilization.

```zig
const js = @import("js");

const ctx = try js.Context.create(allocator);
defer ctx.destroy();

const value = try ctx.evaluate("let x = 40; x + 2");
```

## Use

`zig build` installs `libzig-js.a` and compatible headers under `zig-out/`. See the [Zig API](src/root.zig), [C API](docs/api.md), [timers](docs/timers.md), [WebAssembly and direct-chunk streaming compilation](docs/wasm.md), and [threads/GC](docs/threads/index.md).

## Build And Test

Requires Zig 0.17.0-dev.

```sh
zig build                       # library and headers
zig build test                  # main test root
zig build test262               # configured tc39/test262 corpus
zig build test-c-api            # C and C++ embedding fixtures
zig build benchmark-comparison  # zig-js single/multithread vs JSC
```

Run `zig build --help` for the full command list.

## What Is Not Implemented

- remaining JSC framework/private and Bun/Home ABI surfaces;
- 10 optimizing-JIT PR-249 cases remain blocked; 6 JSC-private or incompatible premises have terminal dispositions ([inventory](docs/.data/pr249-reference-inventory.json));
- automatic/concurrent compaction and a moving nursery;
- additional native optimizer architecture backends. AArch64 executes profiled dynamic arithmetic/comparisons plus unary, bitwise, property, membership, direct-eval, spread, invocation, construction, literal object/array initialization, proper-tail-call effects, and exact normal/throw/return/break/continue completion dispatch through precisely rooted callbacks; see [optimizer status](docs/optimizing-jit.md).

The [release matrix](docs/.data/release-compatibility-matrix.json) tracks [#134](https://github.com/zig-utils/zig-js/issues/134); removal of this section is gated by [#246](https://github.com/zig-utils/zig-js/issues/246).
